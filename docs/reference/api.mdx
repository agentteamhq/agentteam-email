The public API documentation boundary is not finalized.

The repository contains generated and service-local API contracts for validation
and first-party service integration. Those contracts are not a public API
reference until the public support boundary is explicitly accepted and the
owning service publishes a committed specification for it.

When a public API boundary is approved, this page must reference the committed
OpenAPI specification from the owning service contract.
