---
title: "Configuration"
description: "Required configuration surfaces for self-hosted installs."
---

Self-hosted installs must provide configuration explicitly through Helm values,
referenced Secrets or ConfigMaps, and Cloudflare resource configuration.

Published Helm installs use the chart artifact:

```text
oci://ghcr.io/agentteamhq/agentteam-email
```

## Required operator inputs

- public web app URL
- inbound fast-path tunnel URL
- MongoDB connection URLs for app, WildDuck, and control databases
- Redis URL for the mail stack
- R2-compatible archive endpoint, bucket, access key, and secret
- WildDuck admin access token and access-control secret
- mail control API token
- fast-path HMAC secret shared by the Worker and control service
- outbound provider selection: `cloudflare` or `ses`
- Cloudflare account, zone, token, Worker, R2, DNS, and Email Routing settings
- production auth and encryption secrets for the web app

See the Helm self-host guide for concrete variable names and values files.
