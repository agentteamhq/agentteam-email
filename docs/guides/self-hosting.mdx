AgentTeam Email is built for self-hosting. You own the full stack — web app, mail control service, mail servers, archive storage, and Cloudflare integration — running on your infrastructure. Two supported deployment paths let you match the runtime to your environment: Docker Compose for a single host and Helm for Kubernetes. Both paths use the same container images and share the same configuration contract.

## Prerequisites

Both deployment paths share the same external dependencies. Gather these before you begin.

**Cloudflare account and domain**

AgentTeam Email uses Cloudflare for three things: DNS for your mail domain, Email Routing to capture inbound messages via a Cloudflare Email Worker, and R2 object storage for raw message archives. Your domain must use Cloudflare DNS. You need a Cloudflare API token scoped to your account and the zones that receive mail.

**Object storage**

All inbound messages are archived to an R2-compatible bucket before delivery. Cloudflare R2 is the recommended provider, but any S3-compatible endpoint works. Create a bucket and generate access credentials before deployment.

**Domain and hostname**

You need a publicly routable hostname for the web app (for example, `https://mail.company.example`) and a mail domain with Cloudflare Email Routing enabled. The Cloudflare Email Worker posts signed notifications to `<PUBLIC_HOSTNAME>/agent-mail/ingest/v1`, so that path must be reachable from Cloudflare's network.

## Container images

First-party images are published to the GitHub Container Registry under the `agentteamhq/agentteam-email` namespace:

```text
ghcr.io/agentteamhq/agentteam-email/atemail-web-server
ghcr.io/agentteamhq/agentteam-email/atemail-mail-control-service
```

Both deployment paths pull from the same images. The Compose stack defaults to `latest` and can be pinned with `AGENTTEAM_EMAIL_VERSION`. The Helm chart uses the same image references and supports the same pinning approach through chart values.

<Note>
  Third-party service images (MongoDB, Redis, WildDuck, Haraka, ZoneMTA, Rspamd) are pinned directly in the Compose file and Helm chart so dependency update tooling can track them independently.
</Note>
