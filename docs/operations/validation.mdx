After you deploy AgentTeam Email, run through these checks to confirm that every layer — the web app, the fast-path ingest endpoint, and the R2 archive — is operating correctly. You can run all of these checks from any machine that has network access to your deployment.

## Health check

The web app exposes a health endpoint at `/healthz`. A successful response confirms that the web application container is running and reachable:

```bash
curl -i https://mail.company.example/healthz
```

Expect an HTTP `200 OK` response. Any other status code or a connection error indicates a problem with the web app container, your load balancer, or your TLS configuration.

## Fast-path ingest verification

The Cloudflare Worker sends HMAC-signed POST requests to `/agent-mail/ingest/v1` after writing each inbound message to R2. You can confirm that the endpoint is reachable without a valid HMAC signature — an unsigned or unauthenticated request will be rejected, but the rejection itself tells you the endpoint is up:

```bash
curl -i https://mail.company.example/agent-mail/ingest/v1
```

A `405 Method Not Allowed` or `401 Unauthorized` response confirms the endpoint is reachable and the routing is correct. A connection error or `404` means the path is not being served, which indicates a misconfigured reverse proxy or a web app that has not started cleanly.

## Archive verification

After sending a test email to a mailbox on your deployment, verify that the R2 bundle was written correctly.

The inbound bundle path follows this pattern:

```text
orgs/<org_public_id>/domains/<recipient_domain>/mail/inbound/YYYY/MM/DD/<ingest_id>/
```

Check that the following files are present in that prefix:

- `raw.eml` — the original message bytes
- `edge.json` — the Cloudflare Worker envelope metadata and commit marker
- `result.json` — the terminal delivery result

You can inspect these objects using your R2 provider's console or any S3-compatible CLI tool:

<Tabs>
  <Tab title="AWS CLI">
    ```bash
    aws s3 ls s3://agent-mail-archive/orgs/<org_public_id>/domains/<recipient_domain>/mail/inbound/ \
      --endpoint-url https://<account-id>.r2.cloudflarestorage.com \
      --recursive
    ```
  </Tab>
  <Tab title="rclone">
    ```bash
    rclone ls r2:agent-mail-archive/orgs/<org_public_id>/domains/<recipient_domain>/mail/inbound/
    ```
  </Tab>
</Tabs>

If `result.json` is missing, the bundle is incomplete. Check the mail-control-service logs for processing errors and review the Mongo `agent_mail_control` database for `blocked` or `retry_wait` queue items.

<Tip>
  See the [archive layout reference](/reference/archive-layout) for the full file-by-file description of what each object contains and what each status value means.
</Tip>

## Documentation contributor commands

If you are working on the AgentTeam Email documentation source from the repository, the following `mise` tasks are available:

```bash
mise run //docs:dev          # start a local preview server
mise run //docs:validate     # validate documentation structure
mise run //docs:broken-links # check for broken links
mise run //docs:check        # run all documentation checks
```

<Note>
  These `mise` commands are for contributors working on the documentation source. They are not required for deploying or operating AgentTeam Email.
</Note>
