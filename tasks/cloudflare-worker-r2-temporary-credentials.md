# Org-Prefixed Cloudflare Worker Ingest Plan

## Objective

Implement per-domain customer Cloudflare Email Routing Workers that archive
inbound mail into an AgentTeam-owned shared R2-compatible bucket under an
organization-specific prefix.

The web server owns user, organization, Better Auth, Cloudflare OAuth, and
Worker deployment orchestration. The Mail Control Service owns archive storage
contracts, temporary archive credential issuance, runtime domain projection,
queueing, replay, and result processing. The customer Worker is an edge capture
agent only.

The required public flow is:

```text
authenticated user -> web server -> Cloudflare OAuth/account/zone/domain setup
web server -> control service -> org/domain archive prefix and runtime state
web server -> customer Cloudflare account -> one Worker per domain
customer Worker -> AgentTeam-owned R2-compatible bucket under org prefix
customer Worker -> web /rpc/agent-mail/ingest/v1
web -> control service /rpc/agentMail.ingest.enqueue
control service -> queue/replay/result processing
```

## Required Organization Prefix

Every hosted archive object written by a customer Worker must live under an
organization prefix.

Required prefix:

```text
orgs/{org_public_id}/
```

`org_public_id` must be the Better Auth organization public id derived from the
organization UUIDv7 `_id` and encoded as base62. Organization slugs, names,
email domains, user ids, Cloudflare account ids, and Cloudflare zone ids must
not be used as the tenant prefix.

Required inbound bundle shape:

```text
orgs/{org_public_id}/domains/{domain}/mail/inbound/{yyyy}/{mm}/{dd}/{ingest_id}/
  raw.eml
  edge.json
  result.json
  dsn.eml
```

Rules:

- `{domain}` must be the canonical Agent Mail domain for the connection.
- `{ingest_id}` must be a UUIDv7 generated by the Worker.
- Worker credentials must allow writes only under the domain inbound prefix
  `orgs/{org_public_id}/domains/{domain}/mail/inbound/`.
- Control service processing must reject any key whose org prefix does not
  match the persisted org/domain/connection binding.
- Web ingest must reject any Worker notification whose domain or keys do not
  match the active connection's org/domain/prefix binding.

## Boundary Ownership

### Web Server Owns

- Better Auth session and API-key authentication.
- User and organization authorization.
- Cloudflare OAuth grant lifecycle.
- Cloudflare account and zone listing.
- User-driven domain connection creation.
- Per-domain Worker deployment into the selected customer Cloudflare account.
- Worker HMAC secret generation and encrypted storage.
- Installing Worker secrets/configuration into Cloudflare.
- Public Worker ingest authentication at `/rpc/agent-mail/ingest/v1`, with
  `/agent-mail/ingest/v1` kept only as a transition alias for already deployed
  Workers.
- Customer-facing integration status and remediation actions.

### Mail Control Service Owns

- Shared archive bucket configuration.
- Parent R2/S3 credentials.
- Temporary prefix-scoped R2 credential issuance for Workers.
- Canonical archive key builders and parsers.
- Runtime domain registry used by replay, relay, feedback, status, and message
  provenance.
- Inbound queue, R2 sweep, replay, DSN, blocked-item, and result processing.
- Validation that committed archive keys are inside the registered
  org/domain prefix.

### Worker Owns

- Receiving the Cloudflare Email Routing event.
- Writing exact raw RFC822 bytes and `edge.json` with temporary credentials.
- Sending one signed metadata notification to web.
- Nothing else.

The Worker must not call the control service. The browser must not receive
control-service tokens, parent R2 credentials, temporary R2 credentials, Worker
HMAC secrets, or archive encryption keys.

## Credential Model

Parent archive credentials belong to the Mail Control Service boundary.

The control service must expose an internal API for web to request a Worker
credential set for an authorized org/domain/connection. The returned credential
set is for Worker installation only:

- bucket: shared AgentTeam archive bucket
- prefix scope:
  `orgs/{org_public_id}/domains/{domain}/mail/inbound/`
- permission: write required Worker archive objects and read only if required by
  the Worker implementation
- TTL: seven days unless a later Cloudflare limitation requires a shorter value
- values: access key id, secret access key, session token, expiration timestamp

The web server may receive temporary Worker archive credentials from the control
service only to install them into the customer Worker secrets/configuration. Web
must not render them, log them, return them to browsers, or persist decrypted
values outside the secret-installation workflow.

The customer Worker must receive:

```text
AGENTTEAM_ORG_PUBLIC_ID
AGENTTEAM_CONNECTION_ID
AGENTTEAM_DOMAIN_ID
AGENTTEAM_DOMAIN
AGENTTEAM_ARCHIVE_PREFIX
AGENTTEAM_R2_ENDPOINT
AGENTTEAM_R2_BUCKET
AGENTTEAM_R2_ACCESS_KEY_ID
AGENTTEAM_R2_SECRET_ACCESS_KEY
AGENTTEAM_R2_SESSION_TOKEN
AGENTTEAM_R2_CREDENTIAL_EXPIRES_AT
AGENTTEAM_WORKER_HMAC_SECRET
AGENTTEAM_INGEST_URL
```

`AGENTTEAM_R2_ACCESS_KEY_ID`, `AGENTTEAM_R2_SECRET_ACCESS_KEY`, and
`AGENTTEAM_R2_SESSION_TOKEN` must be temporary Worker credentials, not parent
archive credentials.

`AGENTTEAM_WORKER_HMAC_SECRET` signs web ingest notifications. Web stores this
secret encrypted and verifies the Worker request before parsing trusted fields
from the body.

## Existing Cloudflare Schema Changes

`cloudflareConnection` currently mixes connection, domain, and Worker deployment
state. Either migrate it into the records above or keep it as the Cloudflare
account/zone/domain connection record and reference it from
`agentMailDomain`/`agentMailWorkerDeployment`.

Required changes if `cloudflareConnection` remains:

- uniqueness must be org-scoped, not only user-scoped:

```text
{ organizationId, cloudflareAccountId, cloudflareZoneId, domain } unique
```

- the public view must continue to omit `encryptedWorkerHmacSecret` and
  `hmacSecretReference`.
- disconnected connections must clear encrypted Worker HMAC material and must
  trigger runtime sync that disables the org/domain projection.

## Worker Provisioning Flow

1. User authenticates to web.
2. Web resolves active Better Auth organization and verifies the user has
   authority to add or manage mail domains for that organization.
3. User connects Cloudflare OAuth if no active grant exists.
4. User selects Cloudflare account, zone, and domain.
5. Web creates or updates `agentMailDomain` and `cloudflareConnection`.
6. Web computes:

```text
org_public_id = public id for organization._id
archive_prefix = orgs/{org_public_id}/domains/{domain}/mail/inbound
```

7. Web calls `agentMail.runtime.sync` with the org/domain projection.
8. Web calls `agentMail.worker.archiveCredentials.issue`.
9. Web generates or rotates `AGENTTEAM_WORKER_HMAC_SECRET`.
10. Web deploys or updates one Cloudflare Worker for the domain.
11. Web writes Worker bindings/secrets:

```text
AGENTTEAM_ORG_PUBLIC_ID
AGENTTEAM_CONNECTION_ID
AGENTTEAM_DOMAIN_ID
AGENTTEAM_DOMAIN
AGENTTEAM_ARCHIVE_PREFIX
AGENTTEAM_R2_ENDPOINT
AGENTTEAM_R2_BUCKET
AGENTTEAM_R2_ACCESS_KEY_ID
AGENTTEAM_R2_SECRET_ACCESS_KEY
AGENTTEAM_R2_SESSION_TOKEN
AGENTTEAM_R2_CREDENTIAL_EXPIRES_AT
AGENTTEAM_WORKER_HMAC_SECRET
AGENTTEAM_INGEST_URL
```

12. Web configures Cloudflare Email Routing catch-all for the selected zone to
    the Worker.
13. Web marks `agentMailDomain` and `agentMailWorkerDeployment` ready only after
    Worker deploy, Worker secrets, Email Routing, and runtime sync succeed.

## Worker Ingest Flow

1. Worker receives an email event.
2. Worker derives `recipient_domain` from the envelope recipient, not from user
   controlled headers.
3. Worker writes `raw.eml` under:

```text
orgs/{org_public_id}/domains/{domain}/mail/inbound/{yyyy}/{mm}/{dd}/{ingest_id}/raw.eml
```

4. Worker writes `edge.json` under the same bundle prefix.
5. Worker sends metadata-only notification to:

```text
POST {PUBLIC_HOSTNAME}/rpc/agent-mail/ingest/v1
```

6. Worker includes:

```text
X-Agent-Mail-Connection-Id
X-Agent-Mail-Timestamp
X-Agent-Mail-Signature
```

7. Signature input must be:

```text
<timestamp>
<connection-id>
<body>
```

8. Web resolves the connection id, validates active org/domain/deployment state,
   decrypts the Worker HMAC secret, verifies the signature over raw body bytes,
   and only then parses the body.
9. Web validates `organization_public_id`, `archive_prefix`, domain, and key
   shape.
10. Web calls `agentMail.ingest.enqueue` internally.
11. Control service validates the same prefix/domain/key contract and queues the
    bundle.

## Refresh Lifecycle

The web server owns scheduling and Cloudflare secret updates. The control
service owns credential issuance.

Required refresh behavior:

1. Web selects active deployments whose temporary credentials expire within the
   refresh window.
2. Web calls `agentMail.worker.archiveCredentials.issue`.
3. Web updates only Worker secrets/configuration when Worker code is unchanged.
4. Web stores credential expiration metadata and refresh status, not raw
   temporary secrets.
5. If refresh fails but the previous credential is valid for more than 48 hours,
   status becomes `degraded`.
6. If credentials expire within 48 hours, status becomes `expiring`.
7. If credentials are expired, Worker is missing, Email Routing targets the
   wrong Worker, or an archive write probe fails, status becomes `down`.

  using temporary credentials.

## Required E2E Scenarios

### Scenario 1: Single Org, Single Domain, Worker Archive

Flow:

```text
web auth -> org active -> add domain -> fake Cloudflare Worker deploy
-> temporary credentials installed -> Worker writes raw/edge to MinIO
-> Worker notifies web -> web verifies HMAC -> control queues replay
```

Assertions:

- Archive prefix is
  `orgs/{org_public_id}/domains/{domain}/mail/inbound`.
- MinIO objects are under the org/domain bundle path.
- Fake Cloudflare did not receive parent archive credentials.
- Browser-visible responses do not contain temporary credentials or HMAC
  secrets.

### Scenario 2: Two Orgs, Same Domain Attempt

Flow:

```text
org A adds example.test
org B attempts example.test
```

Assertions:

- Product policy decides whether this is allowed.
- If forbidden, org B receives a clear authorization/conflict error.
- If allowed for test-only fake zones, org prefixes still isolate archive
  objects and Worker secrets.

### Scenario 3: Crossed Worker Identity

Flow:

```text
Worker A signs body for org A/domain A
request uses connection id for org B/domain B
```

Assertions:

- Web returns `401` for bad HMAC or unknown/inactive connection.
- Web returns `400` or `403` for authenticated but wrong org/domain/prefix,
  depending on where validation fails.
- Control service is not called when web authentication fails.

### Scenario 4: Prefix Escape Attempt

Flow:

```text
Worker has org A temporary credentials
Worker attempts write under orgs/{org_b_public_id}/...
```

Assertions:

- Fake S3/R2 rejects the write.
- If an escaped key reaches web/control metadata, control rejects enqueue.

### Scenario 5: Credential Refresh

Flow:

```text
deployment near expiration -> web asks control for new temp credentials
-> web updates Worker secrets -> fake Cloudflare records secret-only update
```

Assertions:

- Worker code is not rewritten when only credentials changed.
- New expiration metadata is persisted.
- Failed refresh with valid old credentials results in `degraded`.
- Expired credentials result in `down`.

### Scenario 6: Notification Drop And Sweep Recovery

Flow:

```text
Worker writes raw/edge under org prefix
Worker notification is dropped
R2 sweep discovers bundle
control queues replay
```

Assertions:

- Sweep lists only registered org/domain prefixes.
- Sweep does not process objects from another org prefix.
- Replay remains idempotent.

## Security Acceptance Checks

- Parent archive credentials stay inside Mail Control Service configuration.
- Temporary Worker credentials are never returned to browsers.
- Worker HMAC secrets are encrypted at rest in web DB.
- Public views omit secret-bearing fields.
- Logs and artifacts redact temporary credentials, HMAC secrets, session tokens,
  parent archive credentials, OAuth access tokens, and raw Authorization
  headers.
- Missing, stale, unknown, or invalid Worker credentials fail closed.
- Valid Worker credentials with wrong org/domain/prefix do not enqueue mail.

## Open Questions

- Exact Cloudflare temporary credential API shape and limitations must be
  confirmed against current Cloudflare R2 documentation before implementation.
- If Cloudflare cannot issue the exact prefix-scoped temporary credentials
  required here, the fallback must be an AgentTeam-controlled upload-session API
  that returns presigned object URLs and never exposes parent credentials.
- Product policy for the same domain claimed by two organizations must be
  decided before enforcing the unique index and user-facing error copy.
- Archive object encryption is required by the prior security model, but the
  exact key distribution and Worker encryption mechanism must be specified
  before implementation.
